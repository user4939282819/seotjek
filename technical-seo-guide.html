Technical SEO Guide: Teknik Google Kræver

Technical SEO handler om at få Google til at kunne crawle, indeksere og rank din hjemmeside. Uden teknisk basis fungerer on-page SEO ikke. Denne guide dækker alt fra sitehastighed til Core Web Vitals, mobile-first indexering, og XML sitemaps.

Sitehastighed: Hastighed Er Ranking Faktor

Google måler direkte hvor hurtigt din hjemmeside loader. Langsomme sider rangerer dårlig både på mobil og desktop. Google bruger sitehastighed som direkte ranking signal.

  • Desktop target: Under 2 sekunder på kabelinternet
  • Mobil target: Under 3 sekunder på 4G mobilt netværk
  • Billeder: Komprimér eller brug moderne formater (WebP)
  • Caching: Browser caching betyder browsers gemmer kopier lokalt
  • CDN: Content Delivery Networks gør indhold lokalt tilgængeligt

Core Web Vitals: Brugeroplevesingrammelsed

Google måler brugeroplevelse gennem tre metrics kaldet Core Web Vitals. Hvis du fejler disse, straffer Google dig i ranking.

  • LCP (Largest Contentful Paint): Stort element skal vises på under 2,5 sekunder
  • FID (First Input Delay): Browser skal svare på indput under 100 millisekunder
  • CLS (Cumulative Layout Shift): Siden må ikke rykke rundt mens den loader

Mobile-First Indexering

Google bruger nu primært mobilversionen af hjemmesider til at vurdere indhold og ranking. Hvis din mobil version er dårlig, får du dårlig ranking selv på desktop.

  • Responsivt design: Siden skal se godt ud på alle skærmstørrelser
  • Touch buttons: Buttons skal være mindst 44x44 pixels
  • Tekst læselighed: Tekst skal være læsbar uden zoom
  • Mobil hastighed: Mobil skal være hurtigere end desktop

Crawlability: Lad Google Ind

  • XML Sitemap: Fil der fortæller Google alle dine sider
  • robots.txt: Fil der styrer hvad Google må crawle
  • HTTPS: Sikker forbindelse (næsten obligatorisk nu)
  • Broken links: 404 fejl på vigtige sider skal fikses

Technical SEO er øl-fundamentet. Gør det rigtigt først.
