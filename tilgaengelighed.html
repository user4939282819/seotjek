1. Vores forpligtelse

seotjek.dk, ejet og drevet af NordicWebFlow (CVR: 46305639), er forpligtet til at sikre digital tilgængelighed for alle brugere, herunder personer med syns-, høre-, motoriske eller kognitive funktionsnedsættelser. Vi mener, at alle har ret til at drage nytte af vores gratis SEO-analyseværktøj, uanset evner eller teknologi.

Denne erklæring gælder for alle sider på seotjek.dk, herunder forsiden og SEO-analyseværktøjet, privatlivspolitikken, cookiepolitikken, vilkår og betingelser og denne tilgængelighedserklæring.

2. Overensstemmelsesstandard

Vi stræber efter at opfylde Web Content Accessibility Guidelines (WCAG) 2.1 på niveau AA, som er den internationalt anerkendte standard for webtilgængelighed. WCAG 2.1 er baseret på fire principper:

  • Opfattelig (Perceivable): Information og brugergrænsefladekomponenter skal præsenteres på måder, som brugerne kan opfatte.
  • Anvendelig (Operable): Brugergrænsefladekomponenter og navigation skal kunne betjenes af alle.
  • Forståelig (Understandable): Information og betjening af brugergrænsefladen skal være forståelig.
  • Robust (Robust): Indhold skal være robust nok til at kunne fortolkes af mange forskellige brugeragenter, herunder hjælpeteknologier.

Vi følger desuden EU's webtilgængelighedsdirektiv (2016/2102) og den danske implementering heraf, hvor det er relevant for vores type virksomhed.

3. Tilgængelighedsforanstaltninger

Vi har implementeret følgende konkrete tiltag for at forbedre tilgængeligheden på seotjek.dk:

  • Semantisk HTML-struktur: Korrekt brug af overskrifter (H1-H6 i logisk hierarki), landmærker (nav, main, footer), lister og tabeller med korrekte roller.
  • ARIA-attributter: WAI-ARIA-labels og -roller på alle interaktive elementer, herunder knapper, navigation, dialoger og cookie-banneret.
  • Farvekontrast: Minimum 4.5:1 kontrastforhold mellem tekst og baggrund (AA-niveau) på tværs af alle sider, verificeret med automatiserede værktøjer.
  • Responsivt design: Alle sider fungerer fuldt ud på skærmstørrelser fra 320px til 4K, med touch-venlige interaktionsmål på minimum 44x44 pixels.
  • Tastaturnavigation: Alle interaktive elementer er tilgængelige via tastatur (Tab, Enter, Escape, piletaster). Tydelige fokusindikatorer vises på alle fokusérbare elementer.
  • Alt-tekster: Alle meningsfulde SVG-ikoner har aria-labels. Dekorative elementer er markeret med aria-hidden="true".
  • Læsbar tekst: Alt indhold er skrevet på klart, forståeligt dansk. Tekniske termer forklares i kontekst. Skriftstørrelsen kan skaleres op til 200% uden tab af funktionalitet.
  • Animationer: Vi respekterer prefers-reduced-motion medieforespørgslen — brugere, der foretrækker reduceret bevægelse, vil opleve minimal eller ingen animation.
  • Konsistent navigation: Samme navigationsmønster på alle sider med tydelig struktur og forudsigelige interaktioner.

4. Tilgængelig navigation

Navigationen på seotjek.dk er designet med tilgængelighed som prioritet:

  • Topnavigation: Hovedmenuen er tilgængelig via tastatur med logisk tab-rækkefølge. Dropdown-menuer kan åbnes med Enter og lukkes med Escape.
  • Mobil navigation: Hamburger-menuen er tilgængelig via skærmlæsere med korrekt aria-label og aria-expanded attributter.
  • Footer-navigation: Alle vigtige sider er tilgængelige fra footeren, herunder alle juridiske sider og produktsider.
  • Brødkrummer og links: Tydelig angivelse af den aktuelle side og kontekst via aktive navigationstilstande.
  • Scroll-til-top: En tilgængelig knap med aria-label for hurtigt at vende tilbage til toppen.

5. Indhold og læsbarhed

Vi prioriterer klar og tilgængelig kommunikation:

  • Sproget er sat til lang="da" i HTML-koden, så skærmlæsere korrekt kan udtale dansk tekst.
  • Alle juridiske dokumenter (privatlivspolitik, cookiepolitik, vilkår) er skrevet i klart sprog med indholdsfortegnelser for nem navigation.
  • Sidestruktur følger logisk informationsarkitektur med tydelige overskrifter.
  • FAQ-sektionen bruger accordion-mønsteret med korrekte ARIA-attributter.

6. Kendte begrænsninger

Vi er opmærksomme på, at visse dele af hjemmesiden muligvis ikke er fuldt tilgængelige endnu. Vi arbejder løbende på at forbedre disse:

  • Komplekse datavisualiseringer: Visse diagrammer og grafer i SEO-rapporterne kan være vanskelige at fortolke med skærmlæsere. Vi stræber efter at tilbyde tekstalternativer.
  • Animationer: Selvom vi respekterer prefers-reduced-motion, kan visse overgangseffekter stadig forekomme.
  • Tredjepartsindhold: Eksterne widgets eller indlejret indhold kan have tilgængelighedsbegrænsninger, som vi ikke kontrollerer.
  • PDF-rapporter: Eksporterede PDF-rapporter er muligvis ikke fuldt tilgængelige for skærmlæsere.

7. Teknisk specifikation

seotjek.dk er bygget med følgende tilgængelighedsteknologier:

  • HTML5 med semantiske elementer (header, nav, main, section, article, aside, footer)
  • WAI-ARIA 1.1 attributter for rolledefinitioner, tilstande og egenskaber
  • CSS3 med media queries for responsive design og prefers-reduced-motion, prefers-color-scheme og prefers-contrast respekt
  • Passiv event-lytning for bedre scroll-ydeevne
  • IntersectionObserver API for effektive scroll-animationer (performance-optimeret)
  • GPU-accelererede animationer via translate3d og will-change for jævn ydeevne

Hjemmesiden er testet og fungerer i følgende browsere: Chrome, Firefox, Safari, Edge (seneste 2 versioner) samt iOS Safari og Android Chrome.

8. Testmetoder

Vi tester tilgængeligheden løbende med en kombination af:

  • Automatiserede værktøjer (Lighthouse, axe DevTools, WAVE)
  • Manuel tastaturnavigationstest
  • Skærmlæsertest (VoiceOver, NVDA)
  • Kontrastverifikation med WebAIM Contrast Checker
  • Responsivt design-test på tværs af enheder og skærmstørrelser
